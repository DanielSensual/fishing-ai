# App Store Review Response

Use this when App Review rejects Bite Atlas for Guideline 2.1 because a demo
account was not provided.

## Reply to App Review

```text
Hello App Review,

Bite Atlas does not include account creation, sign-in, login, or any gated
account-based features. No demo account is required to review the app.

All app functionality is available immediately after launch. Reviewers can open
the app, browse the fishing dashboard, regions, spots, species guide, calendar,
gear page, bait shops, rules, and use the Captain AI chat without credentials.

I have also added this clarification to the App Review Information notes.

Thank you.
```

## App Review Information Notes

```text
No demo account is required. Bite Atlas does not include account creation,
sign-in, login, subscriptions, or gated account-based features.

All features are available immediately after launch:
- Fishing dashboard
- Regional condition pages
- Fishing spot pages
- Species guide
- Seasonal calendar
- Gear recommendations
- Bait shop and rules information
- Captain AI chat

To test the Captain AI chat, tap the Captain button and ask a sample question
such as "Should I fish today?" or "Best spot right now?"
```

## Resubmission Checklist

- Add the App Review Information Notes text in App Store Connect.
- Reply to the App Review message with the Reply to App Review text.
- Resubmit the same build if App Store Connect allows it; no new binary is
  needed for this specific issue.
- If App Store Connect requires demo credentials despite the note, enter
  `No login required` or `N/A` in the demo account fields if the UI permits.
